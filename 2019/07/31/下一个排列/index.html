下一个排列

发表于 2019-07-31 | 分类于 leetcode

题目

给出给定序列的下一个字典序排列(常数空间复杂度，在原数组上直接修改)

测试用例

失败原因

遗漏情况，没有考虑全面

思路

惨图如上，太菜了，不过我是打不死的小强！

思路过程如下：

  1. 我只想到先找出相邻元素中后元素大于前元素的坐标，然后交换两元素，其后进行排序，例如[1 3 4 2]->[1 4 3 2]->[1 4 2 3]，但是如果[1 2 4 3]->[1 4 2 3]显然不对，因为[1 3 2 4]才是我们想要的。
  2. 找到相邻元素(i,i+1)后，不应该直接交换，而是应该在[i:]中找出比nums[i-1]大的最小值nums[k]与nums[i]交换，然后对[i+1:]再排序(心想nlogn也行先做出来就行)，nums[i+1:].sort()
  3. 万万没想到nums[i+1:].sort()不管用，后来才意识到nums[i+1:]相当于开辟了新空间，在新空间排序的结果不会影响到nums[]，导致一直错误
  4. 查不到如何在原数组的区间排序就放弃了，看了答案，发现交换元素后，根本不需要做一遍排序，因为nums[i+1:]是逆序的！只需要颠倒就行了！不信，请看[2 5 4 3 1] 找到(2,5)是逆元素对,然后在[5 4 3 1]中找到比2最小的是3，交换[3 5 4 2 1]，恰好逆序，有人可能质疑我故意找容易的例子，其实可以证明的，假设待交换的元素序号为i,比它大的最小元素序号为j，我们能确定[i+1:]一定是逆序的，就是不确定是否交换元素后也逆序，先证明交换前一个元素大于它，nums[j-1]>nums[j]&nums[j]>nums[i]，故nums[j-1]>nums[i]，然后，nums[j:]之后的元素一定都小于nums[i]，想想是不是这么个道理，所以得出交换后依旧逆序。
  5. 万万没想到，由于找最小值时候一个符号，按照我的代码找到的是第一个3，[2,3,1,3,3]->[2 3 3 1 3]->[2 3 3 3 1]，实际答案应该是[2 3 3 1 3]，添了个=号后终于过了，优化的时候发现从后往前找不会出现这个问题

ps：注意考虑[4 3 2 1] ->[1 2 3 4]的情况
